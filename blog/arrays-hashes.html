Ruby Arrays & Hashes

January 21, 2016

This week, I started diving into the programming language of Ruby and one of the more important concepts that I've learned about is the difference between arrays and hashes.

I'll start with arrays. Ruby arrays are basically a list of items grouped together; an array is a collection of data.

One important aspect of arrays to remember is that an array is a number-indexed list. This means that the first item in the array has an index of 0, the second item has an index of 1, the second item has an index of 2, and so on and so forth. Ultimately, the last item in the array will have an index equal to the number of items in the array minus one, which can also be expressed as (array.length - 1).

Additionally, arrays in Ruby can be empty or can contain any type of information inlcuding strings, numbers, and even other arrays. Here is an example of an array:

      cities = ["San Francisco", "San Diego", "Chicago"]
      cities[0] # => "San Francisco"
      cities[2] # => "Chicago"

The array is called cities and it is comprised of four different string elements separated by commas(,) within brackets([]). When you call a certain index on the array, you get the element at the corresponding index returned to you as shown by cities[0] which returned "San Francisco" because it is at the position of index 0.

You can also add new elements to the array:

      cities << "New York"
      cities[3] # => "New York"
                # "New York" was added to the end of the cities array

A hash is also a collection of data, but each element of data is organized by name. A hash is unordered, so instead of assigning elements an index, the data is organized into key/value pairs. I like to think of hashes as a mini dictionary-like list.

      pets = {
        "cats" => 2,
        "dogs" => 1,
        "fish" => 4,
        "rabbits" => 3
      }

      pets["dogs"]      # => 1
      pets["rabbits"]   # => 3

The keys are on the left and are in quotation marks while the values are on the right-side of the rocket(=>) symbol.

You can also easily add a new key and value to the hash, or modify and existing key's corresponding value:

      pets["birds"] = 2       # this is a new key/value pair
      pets["rabbits"] = 5     # the rabbits key already exists

So now the new hash looks like:

      pets = {
        "cats" => 2,
        "dogs" => 1,
        "fish" => 4,
        "rabbits" => 5,       # the value was updated from 3 to 5
        "birds" => 2          # the key/value pair was added
      }

Overall, arrays and hashes have many similar attributes and also share some of the same methods. Although they are similar, deciding which one to use is an important step in writing your program.
