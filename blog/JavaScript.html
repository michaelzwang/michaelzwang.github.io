Ruby Hashes vs JavaScript Objects

February 12, 2016

After spending the past few weeks learning Ruby, it was strange to jump into JavaScript. I immediately noticed many similarities, but it was also hard to work through the differences. In this post I will compare and contrast Ruby hashes to JavaScript objects.

In Ruby, hashes are a way to collect and store information organized in key/value pairs like so:

      hash_name = { key1: value1, key2: value2 }

You can then access a key/value pair like this:

      hash_name[:key1]          #=> value1

In JavaScript however, there is no built-in hash documentation. Rather, you can define an object literal which has similar functionality to a Ruby hash. Instead of a keys, JavaScript has variables called the object properties.

      var new_object = { property1: value1, property2: value2 };

You can access the properties like this:

      new_object.property1;     //=> value1

As you can see, Ruby hashes and JavaScript objects are very similar in appearance and functionality. They are both used to store a collection of data. However, unlike Ruby keys, JavaScript properties must be variables. Additionally, Ruby hashes have built-in methods that do not work in JavaScript.

The biggest and most important difference in my opinion is that JavaScript objects can take a function as a value.

      var person = { name: "Michael"};

      person.say_hi = function(){
        console.log("Hi, my name is " + person.name + "!")
      };

      person.say_hi()

      //=> "Hi, my name is Michael!"

In the example above, I created a person object with a name property of "Michael". I then create a new property say_hi that is a function. I can then call the function on the object!
