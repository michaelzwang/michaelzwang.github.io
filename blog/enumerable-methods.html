Enumerable Methods: group_by

January 27, 2016

This week, I continued to learn more about Ruby and its many built-in methods. One really interesting and useful method is the Enumerable#group_by method. Basically, this method let's us break a collection into groups based on some given criteria. Let's look at some examples!

    group_by { |obj| block }      #=> a_hash

    (1..6).group_by {|i| i%3}     #=> {0=>[3,6], 1=>[1,4], 2=>[2,5]}

The example above shows that the group_by method takes a collection, in this case the numbers 1 - 6 inclusive, and passes them through the block i%3. The method then returns a hash where the keys are the evaluated result from the block and the values are arrays of elements in the collection that correspond to the key. So in the example above, each number 1-6 is modulated by three and then put into a hash. For the key/value pair 0=>[3, 6], 3%3 = 0 and 6%3 = 0.

    pets = ["cat", "dog", "bird", "fish", "turtle"]

    pets.group_by { |pet| pet.length }

        #=>{3=>["cat","dog"], 4=>["bird","fish"], 6=>["turtle"]}

In the code above, we created an array pets containing a string of different types of pets. We then pass the pets array into the block specified by group_by, which then groups the array elements by character length. In the resulting hash, the 3 key contains the elements from pets that are three characters long.

    names = ["Mary", "John", "Matt", "Alex", "Jackie"]

    names.group_by { |name| name[0] }

      #=> {"M"=>["Mary","Matt"], "J"=>["John","Jackie"], "A"=>["Alex"]}

In this example, we are grouping names by their first letter. The group_by method takes the names array and returns a hash with the keys as the first letters and the values as the corresponding names starting with that letter.

Overall, the group_by method is a handy tool to use when you need to efficiently make a hash based on a specific grouping.
